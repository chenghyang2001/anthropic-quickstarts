# Hero 元件規格書

## 概述

Hero 元件作為 HookHub 的主要登陸區塊，設計目標是立即傳達產品的核心價值主張並驅動使用者互動。它結合了引人注目的文案、動態視覺效果與清晰的行動呼籲。

## 目標

1. **吸引注意**：透過動畫與視覺層次，立即抓住訪客目光
2. **傳達價值**：在數秒內清楚說明 HookHub 提供的服務
3. **驅動行動**：引導使用者執行主要操作（瀏覽 Hooks、提交 Hook）
4. **建立信任**：透過數據統計呈現社群證明（可用 hooks 數、下載次數、貢獻者人數）
5. **強化品牌**：維持與 HookHub 色彩系統一致的視覺識別

## 元件結構

### 1. 背景層

裝飾性動畫元素，增添視覺深度而不干擾主內容。

**元素：**
- **漸層光球**：3 個模糊圓形漸層，分佈在角落與邊緣
- **網格圖案**：帶有放射狀遮罩淡出的細緻網格覆蓋層
- **浮動節點**：代表連結性的小型動畫圓點
- **連接線**：使用虛線 SVG 路徑連接各節點

**設計規範：**
- 所有背景元素必須加上 `pointer-events-none`
- 透明度保持低調（光球 ≤30%，連接線 ≤20%）
- 動畫應緩慢且不干擾視線

### 2. 內容區（左欄）

#### 徽章
- 藥丸形狀的標示，帶有脈動圓點
- 傳達核心差異化訊息（例：「社群驅動的自動化工具」）
- 透過細膩動畫呈現活躍感

#### 主標題（H1）
- 多行標題，含強調關鍵字
- 主要關鍵字使用漸層文字效果與底線裝飾
- 字體大小：`4xl`（手機）→ `5xl`（平板）→ `6xl`（桌面）
- 緊湊的行高（1.05）與字距

#### 說明文字
- 1-2 句補充說明標題的文字
- 使用低調顏色（`--slate-light`）
- 字體大小：`lg`（手機）→ `xl`（桌面）

#### 行動呼籲按鈕
- **主要按鈕**：實色背景、箭頭圖示、懸停發光效果
- **次要按鈕**：僅有框線、加號圖示、細緻懸停狀態
- 兩個按鈕均需寬裕的內距與圓角（2xl）
- 懸停狀態包含縮放變換與陰影

#### 數據列
- 橫向展示 3 個關鍵指標
- 以上方框線與行動呼籲區隔
- 大型粗體數字，下方搭配小標籤

### 3. 視覺區（右欄）

代表「hooks」概念的動態插圖。

**元素：**
- **外環**：虛線邊框，緩慢旋轉
- **中環**：實線邊框，反向旋轉
- **內部光暈**：模糊漸層背景
- **中心圖示**：以品牌漸層方框呈現的連結 / hook 圖示
- **軌道元素**：3 個浮動圖示卡，代表各項功能（通知、驗證、速度）

**設計規範：**
- 僅在 `lg` 螢幕以上顯示
- 所有旋轉應流暢且持續
- 軌道元素以不同速度和方向旋轉

## 動畫規範

### 必要動畫
- `animate-fade-in`：透明度 0 → 1
- `animate-slide-up`：Y 軸位移搭配淡入
- `animate-pulse-slow`：緩慢透明度脈動
- `animate-float`：輕柔垂直浮動
- `animate-float-delayed`：延遲的浮動效果
- `animate-ping-slow`：緩慢 ping 效果
- `animate-bounce-slow`：輕柔彈跳
- `animate-spin-slow`：緩慢 360° 旋轉
- `animate-reverse-spin`：逆時針旋轉
- `animate-gradient`：背景位置動畫
- `animate-draw`：SVG 路徑繪製效果
- `animate-dash`：虛線動畫

### 交錯延遲
內容應使用 `animation-delay-*` 類別依序動畫進場：
- 徽章：0ms
- 主標題：0ms（搭配 slide-up）
- 說明文字：200ms
- 行動呼籲按鈕：400ms
- 數據列：600ms

## 響應式行為

| 斷點 | 佈局 | 視覺區 |
|------|------|--------|
| 手機（<768px）| 單欄 | 隱藏 |
| 平板（768px-1023px）| 單欄 | 隱藏 |
| 桌面（≥1024px）| 雙欄 | 顯示 |

### 間距
- 區塊內距：`py-24`（手機）→ `py-32`（桌面）
- 欄間距：欄間 `gap-12`
- 數據列間距：各數據項目間 `gap-8`

## 無障礙設計

- 主標題必須使用 `<h1>` 元素
- 按鈕必須有清晰的焦點狀態
- 動畫應遵守 `prefers-reduced-motion` 媒體查詢
- 所有文字必須有足夠的色彩對比度

## 變體設計規範

建立新的 Hero 變體時：

1. **維持結構**：保留相同的內容層次（徽章 → 主標題 → 說明文字 → 行動呼籲 → 數據列）
2. **保留響應式設計**：複雜視覺效果在手機版一律隱藏
3. **動畫保持低調**：背景動畫絕對不能與主要內容競爭注意力
4. **使用色彩系統**：遵守已定義的顏色或建立互補色彩
5. **確保無障礙**：以螢幕閱讀器與鍵盤導航進行測試
6. **效能考量**：限制 DOM 元素數量，盡量使用 CSS 動畫而非 JavaScript

## Props（未來擴充）

可配置變體的屬性設計建議：

```typescript
interface HeroProps {
  badge?: string;                    // 徽章文字
  headline: {
    lines: string[];                 // 標題各行
    emphasis: string;                // 強調關鍵字
  };
  description: string;               // 說明文字
  primaryCta: {
    label: string;                   // 主要按鈕文字
    href: string;                    // 主要按鈕連結
  };
  secondaryCta?: {
    label: string;                   // 次要按鈕文字
    href: string;                    // 次要按鈕連結
  };
  stats?: Array<{
    value: string;                   // 數據數值
    label: string;                   // 數據標籤
  }>;
  variant?: 'default' | 'minimal' | 'centered' | 'dark';  // 變體類型
}
```

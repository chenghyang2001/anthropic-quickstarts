  <project_name>StudyBuddy — AI-Powered Spaced Repetition Flashcards</project_name>

  <overview>
    StudyBuddy is a PyQt6 desktop flashcard application combining SM-2 spaced repetition
    with Claude AI. Users import Markdown, PDF, or plain text files; Claude auto-generates
    Q&amp;A flashcard pairs from the content. During study sessions, when a card is answered
    incorrectly, Claude provides contextual explanations with root cause analysis and mnemonics.

    The application tracks long-term retention metrics per deck and exports to Anki-compatible
    .apkg format so users can sync with the Anki mobile app.

    Target users: students, language learners, exam preppers, developers learning new domains
    who want AI-assisted card creation and intelligent explanations when they struggle.

    Key differentiators:
    - AI auto-generates flashcards from uploaded documents (no manual card creation)
    - SM-2 spaced repetition for scientifically-optimal review scheduling
    - Claude explains WHY an answer is correct when user rates card 1 or 2
    - Full Anki .apkg export with scheduling data preserved
    - Hierarchical deck organization with tags and bulk operations
  </overview>

  <database_schema>
    <table name="decks">
      <column>id INTEGER PRIMARY KEY AUTOINCREMENT</column>
      <column>name TEXT NOT NULL</column>
      <column>parent_id INTEGER REFERENCES decks(id) ON DELETE CASCADE (NULL = root deck)</column>
      <column>color TEXT DEFAULT '#4A90E2' (hex color shown in sidebar tree)</column>
      <column>created_at DATETIME DEFAULT CURRENT_TIMESTAMP</column>
      <column>description TEXT</column>
      <index>UNIQUE(name, parent_id) — sibling decks cannot share names</index>
      <note>Hierarchy limited to 3 levels (root, child, grandchild) by application logic</note>
    </table>

    <table name="cards">
      <column>id INTEGER PRIMARY KEY AUTOINCREMENT</column>
      <column>deck_id INTEGER NOT NULL REFERENCES decks(id) ON DELETE CASCADE</column>
      <column>front TEXT NOT NULL (question or cloze template)</column>
      <column>back TEXT NOT NULL (answer or cloze fill)</column>
      <column>card_type TEXT DEFAULT 'basic' CHECK(card_type IN ('basic', 'cloze', 'image'))</column>
      <column>tags TEXT DEFAULT '' (comma-separated tag string)</column>
      <column>interval INTEGER DEFAULT 1 (SM-2: days until next review)</column>
      <column>ease_factor REAL DEFAULT 2.5 (SM-2: difficulty multiplier, min 1.3)</column>
      <column>repetitions INTEGER DEFAULT 0 (SM-2: consecutive correct reviews)</column>
      <column>next_review DATE DEFAULT (date('now')) (YYYY-MM-DD)</column>
      <column>ai_explanation TEXT (Claude's explanation, cached after first "Why Wrong?")</column>
      <column>suspended BOOLEAN DEFAULT 0 (suspended cards skip scheduling)</column>
      <column>created_at DATETIME DEFAULT CURRENT_TIMESTAMP</column>
    </table>

    <table name="study_sessions">
      <column>id INTEGER PRIMARY KEY AUTOINCREMENT</column>
      <column>started_at DATETIME NOT NULL</column>
      <column>ended_at DATETIME</column>
      <column>cards_reviewed INTEGER DEFAULT 0</column>
      <column>avg_rating REAL (average quality rating 1-5 for this session)</column>
      <column>deck_id INTEGER REFERENCES decks(id) ON DELETE SET NULL</column>
    </table>

    <table name="card_reviews">
      <column>id INTEGER PRIMARY KEY AUTOINCREMENT</column>
      <column>card_id INTEGER NOT NULL REFERENCES cards(id) ON DELETE CASCADE</column>
      <column>session_id INTEGER REFERENCES study_sessions(id) ON DELETE SET NULL</column>
      <column>rating INTEGER NOT NULL CHECK(rating BETWEEN 1 AND 5)</column>
      <column>reviewed_at DATETIME DEFAULT CURRENT_TIMESTAMP</column>
      <column>time_spent INTEGER (seconds spent on this card before rating)</column>
    </table>
  </database_schema>

  <core_features>
    <feature id="1" name="Note Import and AI Flashcard Generation">
      Accept .md, .pdf, .txt files via QFileDialog or drag-and-drop (QDropEvent).
      Text extraction: pdfplumber for .pdf, markdown2 for .md, plain read for .txt.
      Chunk extracted text into ~500-token segments (split on paragraph boundaries).
      Send each chunk to Claude (claude-sonnet-4-6) with the generation prompt.
      Claude returns JSON: [{"question": "...", "answer": "...", "type": "basic|cloze"}].
      Cloze auto-detection: if answer appears verbatim in question context, convert to cloze format.
      CardPreviewDialog: editable QTableWidget showing generated cards before saving.
      User can delete, edit, or change card type before committing to DB.
      Generated cards saved to selected deck after user confirmation.
    </feature>

    <feature id="2" name="SM-2 Spaced Repetition Scheduler">
      Full SM-2 algorithm in sm2.py as a pure function (no DB dependency):
        calculate_next_review(interval, ease_factor, repetitions, rating) -> (new_interval, new_ease, new_reps, next_date)
      Rating scale: 1=Again(blackout), 2=Hard, 3=Good, 4=Easy, 5=Perfect
      SM-2 rules:
        - rating >= 3: repetitions += 1, new_interval based on repetitions count
        - rating < 3: repetitions = 0, interval = 1 (reset)
        - ease_factor += 0.1 - (5-rating) * (0.08 + (5-rating) * 0.02)
        - ease_factor clamped to [1.3, ∞)
        - interval_1 = 1, interval_2 = 6, interval_n = round(prev_interval * ease_factor)
      Overdue-first queue: cards sorted by (next_review ASC, id ASC).
      Configurable new cards per day (default 20, stored in local settings).
    </feature>

    <feature id="3" name="Multiple Card Types">
      Basic (default): front = question text, back = answer text. Card flips on Space.
      Cloze deletion: front uses {{c1::hidden}} syntax, displayed as blank "____" in study mode.
        Back shows the full sentence with the cloze word highlighted.
      Image occlusion: placeholder type — front shows "Image card (not yet supported)",
        back shows the answer text. Stored in DB as card_type='image'.
      Bulk type conversion: select multiple cards in CardListWidget → right-click → Convert to Cloze.
      Cloze conversion: wraps the selected answer text in {{c1::}} within the front text.
    </feature>

    <feature id="4" name="Daily Study Session">
      StudyWindow: full-screen QWidget (no parent, no title bar chrome).
      Session start: create study_sessions record, load due cards (sorted by next_review).
      Card display: front text in large QLabel centered in QFrame.
      Flip: Space bar or "Show Answer" button → reveal back text below with QPropertyAnimation.
      Rating buttons (1-5) appear only after flip. Keyboard shortcut: 1-5 keys.
      On rating: call sm2.calculate_next_review() → update card in DB → log card_reviews record.
      Undo last rating: revert card to previous SM-2 state (store pre-rating snapshot in memory).
      Session summary dialog: total reviewed, avg rating, time spent, streak update.
      Streak tracking: consecutive calendar days with at least one session having avg_rating >= 3.
    </feature>

    <feature id="5" name="AI Why Was I Wrong Explanation">
      Triggered when user rates a card 1 or 2 (incorrect/hard) in study session.
      Button "💬 Why Wrong?" appears below the answer after a low rating.
      Claude prompt: "Explain why [front] has answer [back]. Provide: 1) root cause of confusion,
        2) a memorable mnemonic, 3) a concrete real-world example. Be concise (max 150 words)."
      Response streams into an ExplanationPanel (QFrame that slides in below the rating buttons).
      Rate-limited: max 10 AI explanations per study session (counter shown in panel header).
      Explanation cached in cards.ai_explanation (reused on next incorrect rating — no repeat API call).
      Uses claude-sonnet-4-6 for higher-quality explanations.
    </feature>

    <feature id="6" name="Deck Organization">
      Hierarchical deck tree: QTreeWidget with 3 levels max (root/child/grandchild).
      Context menu on right-click: Add Child Deck, Rename, Change Color, Delete (with confirmation).
      Color picker: QColorDialog for deck label color.
      Card tagging: comma-separated tags stored in cards.tags, filterable in card list.
      Full-text search: QLineEdit filters card list by front/back/tags text (case-insensitive).
      Bulk operations on selected cards (multi-select QTableWidget):
        - Move to deck (submenu of all decks)
        - Suspend / Unsuspend
        - Delete (with count confirmation)
        - Change tags
    </feature>

    <feature id="7" name="Study Statistics">
      StatsWidget embedded as a tab in the main content area.
      Retention rate per deck: percentage of reviews with rating >= 3 over last 30 days.
      Streak calendar heatmap: custom QWidget.paintEvent() drawing 7-wide grid of day cells,
        colored by cards-reviewed count (0=white, light/medium/dark green scale).
      Cards learned per day: matplotlib FigureCanvas bar chart (new cards reaching repetitions=1).
      Ease factor trend: line chart of average ease_factor per deck over time.
      CSV export: write study_sessions and card_reviews data to CSV with column headers.
    </feature>

    <feature id="8" name="Anki Export">
      genanki library for .apkg format generation.
      Map decks hierarchy to Anki format: Parent::Child::Grandchild name strings.
      Basic cards → genanki.Note with Basic model (Front/Back fields).
      Cloze cards → genanki.Note with Cloze model (Text field with {{c1::}} preserved).
      Include scheduling data: interval, ease_factor, repetitions mapped to Anki card properties.
      Save .apkg to ~/Documents/StudyBuddy/exports/DeckName_YYYYMMDD.apkg.
      Show success dialog with file path and open-folder button.
    </feature>
  </core_features>

        generate_cards(chunks: list[str]) → list[dict] (async-compatible QThread call)
        prompt: "You are a flashcard expert. Generate Q&amp;A flashcard pairs from this text.
          Return JSON array only: [{\"question\": \"...\", \"answer\": \"...\", \"type\": \"basic\"}]
          Generate 2-5 cards per chunk. Focus on key facts, definitions, and relationships."

      tests/test_import.py: text extraction for each file type, chunking logic, JSON validation.
    </step>

      tests/test_sm2.py: ensure calculate_next_review() handles all rating values correctly.
